## v4.0 (текущая)

**Статус:** АКТИВНА  
**Язык инструкции:** English (ответы — на языке пользователя, правило 1.4)  
**База:** INSTRUCTION_v10.2_light_beta + Philosophical_instruction_v3.5_beta  

### Добавлено
- **Domain Detection Protocol (3.2)** — при обнаружении узкой специализации агент запрашивает доменные критерии (метрики, пороги тревоги, известные ловушки) до выдачи ответа. Без них — все доменные утверждения 🟡, пометка «⬛ domain criteria not provided».
- **Anti-Sycophancy: long-session fix (3.16)** — при возврате к теме после >5 сообщений агент уточняет: «Это то же утверждение или появились новые данные?» — до применения Anti-Sycophancy или обновления статуса.
- **DeepSeek thinking block guidance (1.1)** — явные правила: свободные рассуждения внутри thinking-блока разрешены, структуры pipeline (ШАГ N, TOOLS:, чеклисты) — запрещены.
- **Epistemic limitation note in onboarding (0)** — явное предупреждение: инструкция обеспечивает эпистемическую честность, но не доменную экспертизу; для специализированных задач пользователь должен предоставить критерии.
- **Forbidden: correlation as causation (8)** — явный запрет интерпретировать корреляцию как причинно-следственную связь без оговорок.
- **Think Pipeline расширен до 8 шагов** — добавлен Шаг 5 (Domain Detection) между амбигуацией и протоколами.
- **Domain Detection добавлен в чек-лист (5)** — базовый чек-лист теперь включает проверку доменного запроса.
- **Domain Detection добавлен в Quick Reference (9.1)** — Block 2 маршрутизации включает шаг 6 (Specialized domain?).

### Удалено
- Шапка метаданных (дата, статус, архитектура, список изменений) — убраны из основного файла.
- **Appendix B: Glossary** — вынесен в отдельный файл `GLOSSARY.md`.
- **Appendix C: Migration Protocol for Developers** — полностью вырезан.
- **Section 8: Examples** — убраны для сокращения размера инструкции. При необходимости примеры подаются как few-shot в запросе.

### Изменено
- Инструкция переведена на английский для экономии токенов (~30%).
- Правило языка ответа (1.4) сохранено без изменений.
- Название: «Philosophical Instruction v4» (без версии-ветки, без даты).

## v3.5 Beta (Philosophical_instruction_v3_5_beta)

**Статус:** АРХИВ  
**Язык:** Русский  

### Ключевые особенности
- Философское ядро (Пирс, Фейнман, Канеман, Пойа) как отдельная Часть I.
- Think Pipeline реализован через вопросы, а не явные шаги.
- Артефакты из ветки 10.2: Карантин Контекста, Anti-Sycophancy, CoVe/ToT/Pre-Mortem/Self-Consistency/Contrastive Check, 7 Грехов, Red Teaming, Опасные Зоны, управление контекстом.
- Онбординг: явная проверка окружения + режимы + светофоры.
- Блок 6 «Телос» — тест «Если бы пользователь видел всё что я знаю — принял бы он то же решение?»

### Известные ограничения (выявлены в тестировании PG_EXPECTO)
- Нет механизма запроса доменных критериев → агент может присвоить 🟡 там где нужно 🔴.
- Предупреждение о корреляции/причинности не формализовано.
- Большой размер → превышение токенного лимита при работе с объёмными входными данными.
