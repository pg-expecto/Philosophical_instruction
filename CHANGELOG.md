## v5.0 (текущая)

**Статус:** АРХИВ  
**Язык инструкции:** English (ответы — на языке пользователя, правило 1.4)  
**База:** Philosophical_instruction_v4 → структурный аудит → исправление 12 архитектурных проблем  

### Структурные исправления

- **Think Pipeline: 8 → 7 шагов.** Domain Detection и Ambiguity объединены в STEP 2 вместе с классификацией. Устранена тройная точка входа Domain Detection (была в pipeline STEP 5, в разделе 3.2, в маршрутизации BLOCK 2 п.6). Теперь — единственная точка вызова в STEP 2.
- **Протокол возврата после уточнений.** Добавлено явное правило: «On return after clarification: re-enter pipeline at STEP 2 with updated context.» В v4 агент не знал, с какого шага продолжать после ответа пользователя на уточняющий вопрос.
- **Раздел 3.2 разделён на два.** Domain Detection (3.2) и Clarifying Questions (3.3) — теперь отдельные разделы с разными триггерами. Общее правило: все вопросы комбинируются в один раунд ≤3. В v4 два протокола с разными триггерами жили в одном разделе и могли породить 6 вопросов (3+3).
- **Таблица протоколов (1.1) расширена.** Добавлены Debug, Review, Analysis/Recommendation, Brainstorming. В v4 таблица покрывала 6 из 9 типов задач; в v5 — все 10. Дополнительные протоколы (Isolation, MRE, Review Scale, 7 Sins) вынесены в примечание под таблицей.
- **Чек-листы (раздел 5) расширены.** Добавлены чек-листы для: Debug (Isolation + MRE + 7 Sins), Review (Review Scale + Red Teaming + итоговый светофор), Analysis/Recommendation (Partial Knowledge + допущения + Contrastive Check), Brainstorming (маркировка ⬛/🔴 + заявление режима + Anti-Sycophancy off). В v4 чек-листы были только для Base, Code, High-Stakes, Long chat.

### Добавлено

- **Brainstorming exit rule (3.23).** Brainstorming завершается когда сообщение пользователя не содержит триггеров И классифицируется как другой тип задачи. При неясности — агент спрашивает. В v4 не было механизма выхода — агент мог бесконечно оставаться в спекулятивном режиме.
- **Contrastive Check depth limit (3.14 п.4).** Явное правило: «Apply once per conclusion. Do not recursively check the counterargument itself.» В v4 рекурсия не ограничена — потенциально бесконечный цикл внутри think pipeline.
- **Contrastive Check расширен на Analysis/Recommendation.** Теперь рекомендован не только для Architecture и High-Stakes, но и для Analysis. Отражено в таблице протоколов, чек-листе и Quick Reference.
- **Context Migration trigger (3.15).** Добавлена связка с Appendix A: «Context Migration between chats: See Appendix A. Trigger: user explicitly starts a new chat to continue a previous task, or provides a migration summary.» В v4 Appendix A висел без точки входа из основного тела инструкции.
- **Appendix A trigger.** Добавлено явное условие вызова: «User explicitly starts a new chat to continue a previous task, provides a migration summary, or says 'continuing from previous chat'.»

### Изменено

- **Priority 2 (Safety) расширен.** Было: «Code must withstand attack (Red Teaming).» Стало: «Code must withstand attack (Red Teaming). Danger Zones Type A (4.1) require specialist referral. No harmful advice in medicine, law, finance, security.» В v4 Priority 2 покрывало только безопасность кода, но не Danger Zones Type A.
- **Нумерация разделов сдвинута** из-за разделения 3.2 на два раздела. Source Trust Hierarchy: 3.6→3.7. Ambiguity: 3.7→3.8. CoVe: 3.8→3.9. И т.д. Все внутренние ссылки (Forbidden Responses, Quick Reference, чек-листы) обновлены.
- **Quick Reference (9.1) синхронизирован.** BLOCK 2 п.4 теперь объединяет Ambiguity + Domain + Clarifying Questions в одну проверку. Убран отдельный пункт Domain Detection. BLOCK 3 п.4 ссылается на расширенную таблицу протоколов (включая Isolation, MRE, Review Scale). Quick Reference таблица (9.2) включает все 15 протоколов с актуальными номерами разделов.
- **Ideal Response Architecture (раздел 7)** обновлена под 7 шагов. STEP 2 явно включает Classifier + Ambiguity + Domain. STEP 4 перечисляет все протоколы включая Isolation/MRE/Review Scale.

### Известные ограничения (не закрыты в v5, кандидаты на v6)

- **Нет адаптивной сложности.** Протоколы применяются бинарно. Нет degradation strategy при исчерпании контекстного окна.
- **Нет протокола самокоррекции.** Belief Update (3.18) реагирует на данные от пользователя. Нет механизма когда агент сам обнаруживает ошибку в предыдущем ответе.
- **Нет бюджета раундов уточнений.** Максимум 3 вопроса за раунд, но количество раундов не ограничено.
- **Нет обнаружения конфликтов в требованиях пользователя.** Context Management (3.15) ловит drift по длине чата, но не по семантическому конфликту (PostgreSQL в сообщении 3, MongoDB в сообщении 15).
- **Нет весов протоколов.** При ограниченном think pipeline неизвестно какие протоколы приоритетнее.
- **Нет философского нарратива.** Инструкция остаётся процедурной. По рекомендации К.В. Воронцова: добавить слой обоснований (научный метод, формы логического вывода, когнитивные искажения, этический фундамент). Объём инструкции ~15 стр. vs рекомендуемые ~50.

## v4.0

**Статус:** АРХИВ  
**Язык инструкции:** English (ответы — на языке пользователя, правило 1.4)  
**База:** INSTRUCTION_v10.2_light_beta + Philosophical_instruction_v3.5_beta  

### Добавлено
- **Domain Detection Protocol (3.2)** — при обнаружении узкой специализации агент запрашивает доменные критерии (метрики, пороги тревоги, известные ловушки) до выдачи ответа. Без них — все доменные утверждения 🟡, пометка «⬛ domain criteria not provided».
- **Anti-Sycophancy: long-session fix (3.16)** — при возврате к теме после >5 сообщений агент уточняет: «Это то же утверждение или появились новые данные?» — до применения Anti-Sycophancy или обновления статуса.
- **DeepSeek thinking block guidance (1.1)** — явные правила: свободные рассуждения внутри thinking-блока разрешены, структуры pipeline (ШАГ N, TOOLS:, чеклисты) — запрещены.
- **Epistemic limitation note in onboarding (0)** — явное предупреждение: инструкция обеспечивает эпистемическую честность, но не доменную экспертизу; для специализированных задач пользователь должен предоставить критерии.
- **Forbidden: correlation as causation (8)** — явный запрет интерпретировать корреляцию как причинно-следственную связь без оговорок.
- **Think Pipeline расширен до 8 шагов** — добавлен Шаг 5 (Domain Detection) между амбигуацией и протоколами.
- **Domain Detection добавлен в чек-лист (5)** — базовый чек-лист теперь включает проверку доменного запроса.
- **Domain Detection добавлен в Quick Reference (9.1)** — Block 2 маршрутизации включает шаг 6 (Specialized domain?).

### Удалено
- Шапка метаданных (дата, статус, архитектура, список изменений) — убраны из основного файла.
- **Appendix B: Glossary** — вынесен в отдельный файл `GLOSSARY.md`.
- **Appendix C: Migration Protocol for Developers** — полностью вырезан.
- **Section 8: Examples** — убраны для сокращения размера инструкции. При необходимости примеры подаются как few-shot в запросе.

### Изменено
- Инструкция переведена на английский для экономии токенов (~30%).
- Правило языка ответа (1.4) сохранено без изменений.
- Название: «Philosophical Instruction v4» (без версии-ветки, без даты).
